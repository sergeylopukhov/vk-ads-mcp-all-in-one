## Сообщества VK

Нужен отдельный доступ Core VK API с правами `groups` и `wall`. При установке выберите функцию сообществ и legacy OAuth: установщик откроет браузер и попросит полный URL страницы `oauth.vk.ru/blank.html`. Встроенное приложение предлагается по умолчанию; вместо него можно указать своё с нужными доступами. Для ранее настроенных приложений доступен VK ID OAuth. Инструменты не читают списки участников и не возвращают полные тексты постов.

Для обычного поиска используйте `vk_start_community_research` (или совместимый `vk_research_communities`): он сохраняет полный отфильтрованный список кандидатов, затем анализирует его в фоне пакетами по 25. Пока анализ идёт, сервер отправляет стандартное MCP-уведомление с безопасными счётчиками раз в минуту и отдельное уведомление с итогом. Без ручной настройки применяется универсальный профиль: нормированные совпадения в названии, описании и постах, свежесть, доля тематических публикаций, штрафы за низкую тематичность и низкую активность, исключающие термины. Результат делится на `passed` (по умолчанию от 60), `review` (45–59) и `rejected`; вычисляемые флаги риска формируются вместе с баллом. Неполные `scoring_rules` переопределяют лишь явно заданные значения, а не выключают остальные сигналы. `vk_get_community_research_progress` остаётся запасным способом прочитать ход обработки, а `vk_get_community_research_run` возвращает готовый снимок без нового запроса к VK. Поисковые и включающие фразы подтверждаются в названии либо описании; тип, география, размер и исключающие термины отсекаются до чтения стены. `limit` нужен только для явного ограничения объёма работы. При `min_members` выдача VK сначала запрашивается по числу участников; пустой ответ на такой сортировке автоматически повторяется по релевантности. Если VK или явно заданный лимит не позволили охватить всё, ответ содержит `summary.incomplete` и причины. Экспорт создаётся в памяти, помечает строки `pending_approval` и ничего не добавляет в сегменты.

| Инструмент | Доступ | Описание | Проверено |
| --- | --- | --- | --- |

| `vk_research_communities` | `read` | Совместимый alias фонового запуска с теми же уведомлениями. | ✅ |

`vk_score_communities` принимает `terms`, `exclude_terms`, объекты `weights`, `per_match_weights` и `term_weights`, `activity_fresh_days`, `min_posts_per_week`, `min_thematic_post_share`, `members_range`, `min_score` и `review_min_score`. `weights` задаёт потолок каждого сигнала; `per_match_weights` — вклад одного совпадения; `term_weights` повышает вклад отдельных профессиональных терминов. Поддерживаемые сигналы: `name_term`, `description_term`, `post_term`, `activity_fresh`, `activity_low_penalty`, `thematic_post_share`, `thematic_low_penalty`, `members_range`, `exclude_term_penalty`. Кластер дополнительно может задать `match_mode`, минимальную тематичность и активность, `require_no_risk_flags` или конкретные `exclude_risk_flags`. Нужен хотя бы один положительный вес; для весов совпадения нужен непустой `terms`. Неизвестные поля отклоняются с ошибкой.

## Служебные инструменты MCP

Служебные вызовы нужны для поиска, проверки режима и подтверждения записи. Они не входят в число 166 возможностей каталога.

| Инструмент | Когда нужен | Проверено |
| --- | --- | --- |
| `get_provider_context` | Показывает активный профиль, connection ID, режим и фиксированный адрес VK Ads API. Первый вызов в новой сессии. | ✅ |
| `account_profile_get` | Возвращает профиль и connection ID, выбранные при запуске. | ✅ |
| `auth_check` | Проверяет авторизацию и возвращает сводку кабинета без токенов и email. | ✅ |
| `search_tools` | Ищет возможности по названию и категории; при необходимости показывает `planned`. | ✅ |
| `call_read_tool` | Вызывает разрешённый read-only alias через единый allowlist. | ✅ |
| `write_capabilities` | Объясняет, доступен ли режим записи и какие ограничения действуют. | ✅ |
| `write_preview` | Проверяет payload и создаёт краткоживущий preview без записи в VK Ads. | ✅ |
| `write_execute` | Выполняет один свежий preview; по умолчанию после точного подтверждения пользователя. | ✅ |
| `write_audit_list` | Возвращает обезличенный журнал preview и выполнений текущего процесса. | ✅ |
| `media_validate` | Проверяет локальный файл до подготовки загрузки. | ✅ |
| `creative_preflight` | Проверяет package, content IDs и текстовые блоки креатива. | ✅ |

## Рекомендуемый порядок работы

1. Вызвать `get_provider_context` и проверить профиль и режим.
2. Найти нужную возможность через `search_tools`.
3. Перед изменением прочитать объект по точному ID.
4. Для записи выполнить подходящий preflight и получить preview.
5. Передать preview в `write_execute` после явного подтверждения пользователя, если оно не отключено владельцем локального профиля.
6. Перечитать изменённый объект и проверить его итоговый статус.

Универсального raw-request инструмента нет. Каждый вызов использует фиксированный API host, проверенную схему и разрешённый путь VK Ads.
