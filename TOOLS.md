Каталог соответствует `mcp-server/src/tool-catalog.ts`. Здесь собраны публичные названия из `search_tools`, без низкоуровневых обёрток. В режиме `readonly` инструменты записи не регистрируются.

> [!IMPORTANT]
> Инструменты записи сначала проверяют параметры и создают preview. `write_execute` выполняет операцию после одноразового подтверждения пользователя. Владелец профиля может отключить это требование через `VK_ADS_REQUIRE_WRITE_CONFIRMATION=0`. Тестовые вызовы ограничены объектами с префиксом `__MCP_TEST__` и списками разрешений.

## Как читать каталог

| Обозначение | Значение |
| --- | --- |
| `✅` | Инструмент реализован и проверяется каталогом и контрактами репозитория. |
| `⛔️` | Инструмент запланирован или его контракт ещё не подтверждён; вызвать его нельзя. |
| `read` | Читает данные VK Ads или выполняет локальный расчёт без изменения кабинета. |
| `write` | Готовит операцию записи; выполнение проходит через preview и подтверждение. |

## Сообщества VK

Нужен отдельный доступ Core VK API с правами `groups` и `wall`. При установке выберите функцию сообществ и legacy OAuth: установщик откроет браузер и попросит полный URL страницы `oauth.vk.ru/blank.html`. Встроенное приложение предлагается по умолчанию; вместо него можно указать своё с нужными доступами. Для ранее настроенных приложений доступен VK ID OAuth. Инструменты не читают списки участников и не возвращают полные тексты постов.

Обычная последовательность: `vk_discover_communities` → `vk_analyze_communities` → `vk_score_communities` → `vk_export_community_candidates`. Все четыре инструмента работают только на чтение. Экспорт создаётся в памяти, помечает строки `pending_approval` и ничего не добавляет в сегменты.

| Инструмент | Доступ | Описание | Проверено |
| --- | --- | --- | --- |
| `vk_discover_communities` | `read` | Ищет по `keywords`, дедуплицирует ID, дополняет metadata и применяет фильтры по терминам, типу, географии и числу участников. | ✅ |
| `vk_analyze_communities` | `read` | Анализирует metadata и последние публичные записи по `community_ids`; указывает активность и риски без сохранения полного текста. | ✅ |
| `vk_score_communities` | `read` | Прозрачно считает от 0 до 100 по `scoring_rules`, показывает причины, штрафы, риски и кластеры. | ✅ |
| `vk_export_community_candidates` | `read` | Формирует CSV или JSON из результатов поиска, анализа и скоринга со статусом `pending_approval`. | ✅ |

`vk_score_communities` принимает `terms`, `exclude_terms`, объект `weights`, `term_weights`, `activity_fresh_days`, `min_posts_per_week`, `min_thematic_post_share`, `members_range` и `min_score`. Поддерживаемые веса: `name_term`, `description_term`, `post_term`, `activity_fresh`, `activity_low_penalty`, `thematic_post_share`, `members_range`, `exclude_term_penalty`. `term_weights` позволяет повысить вклад отдельных профессиональных терминов. Нужен хотя бы один положительный вес; для весов совпадения нужен непустой `terms`. Неизвестные поля отклоняются с ошибкой.

## Служебные инструменты MCP

Служебные вызовы нужны для поиска, проверки режима и подтверждения записи. Они не входят в число 166 возможностей каталога.

| Инструмент | Когда нужен | Проверено |
| --- | --- | --- |
| `get_provider_context` | Показывает активный профиль, connection ID, режим и фиксированный адрес VK Ads API. Первый вызов в новой сессии. | ✅ |
| `account_profile_get` | Возвращает профиль и connection ID, выбранные при запуске. | ✅ |
| `auth_check` | Проверяет авторизацию и возвращает сводку кабинета без токенов и email. | ✅ |
| `search_tools` | Ищет возможности по названию и категории; при необходимости показывает `planned`. | ✅ |
| `call_read_tool` | Вызывает разрешённый read-only alias через единый allowlist. | ✅ |
| `write_capabilities` | Объясняет, доступен ли режим записи и какие ограничения действуют. | ✅ |
| `write_preview` | Проверяет payload и создаёт краткоживущий preview без записи в VK Ads. | ✅ |
| `write_execute` | Выполняет один свежий preview; по умолчанию после точного подтверждения пользователя. | ✅ |
| `write_audit_list` | Возвращает обезличенный журнал preview и выполнений текущего процесса. | ✅ |
| `media_validate` | Проверяет локальный файл до подготовки загрузки. | ✅ |
| `creative_preflight` | Проверяет package, content IDs и текстовые блоки креатива. | ✅ |

## Рекомендуемый порядок работы

1. Вызвать `get_provider_context` и проверить профиль и режим.
2. Найти нужную возможность через `search_tools`.
3. Перед изменением прочитать объект по точному ID.
4. Для записи выполнить подходящий preflight и получить preview.
5. Передать preview в `write_execute` после явного подтверждения пользователя, если оно не отключено владельцем локального профиля.
6. Перечитать изменённый объект и проверить его итоговый статус.

Универсального raw-request инструмента нет. Каждый вызов использует фиксированный API host, проверенную схему и разрешённый путь VK Ads.
