# VK Ads MCP All in One

Локальный MCP-сервер для работы с VK Ads API. Он запускается на компьютере пользователя, подключается к его рекламному кабинету и по умолчанию ничего не меняет.

Сервер использует стандартный транспорт `stdio`. Его можно подключить к Codex, Claude Code, Gemini CLI, Qwen Code, Kimi Code CLI и любому другому клиенту, который поддерживает MCP-инструменты.

## Что умеет сервер

- читать рекламные планы, кампании, группы, баннеры, аудитории, пакеты размещения и справочники;
- получать статистику, сравнивать периоды, ранжировать кампании по CTR, CPC, CPA и расходу, находить аномалии и проблемы открутки;
- готовить отчёты в CSV и XLSX без записи файлов на диск;
- проверять изображения, видео, HTML5-архивы, тексты и параметры объявления до отправки в VK Ads;
- работать с лид-формами, сегментами, ремаркетингом, агентскими кабинетами, мобильными приложениями и асинхронными отчётами в пределах подтверждённых контрактов API;
- выполнять только изолированные тестовые записи через предварительный просмотр и отдельное подтверждение.

Полный перечень возможностей возвращает инструмент `search_tools`.

## Установка

Нужен Node.js 20 или новее.

```bash
git clone <адрес-вашего-репозитория> vk-ads-mcp
cd vk-ads-mcp
npm ci
npm run build
```

## Подключение к AI-клиенту

В клиенте укажите команду `node`, абсолютный путь к `dist/index.js` и профиль. Большинство клиентов с JSON-настройками используют такую форму:

```json
{
  "mcpServers": {
    "vk-ads": {
      "command": "node",
      "args": ["/абсолютный/путь/vk-ads-mcp/dist/index.js"],
      "env": {
        "VK_ADS_PROFILE": "default"
      }
    }
  }
}
```

Путь к `dist/index.js` должен быть абсолютным. Не добавляйте в конфигурацию клиента токен, секрет приложения или пароль от хранилища секретов.

После подключения начните с запроса:

```text
Покажи контекст подключения VK Ads и доступные рекламные планы. Ничего не меняй.
```

## Доступ к VK Ads

Репозиторий не содержит токенов, секретов приложений, персональных данных и рабочего хранилища секретов. Переменные запуска перечислены в [`.env.example`](.env.example). Это справочный файл: Node.js не читает `.env` автоматически.

Сервер поддерживает три способа доступа:

1. Токен VK Ads в окружении запуска `VK_ADS_TOKEN`.
2. `VK_ADS_CLIENT_ID` и `VK_ADS_CLIENT_SECRET` собственного приложения VK Ads для получения и обновления токена.
3. Браузерное подключение через `vk_ads_oauth_begin`, если для приложения уже зарегистрирован точный локальный адрес обратного вызова.

Для публичного GitHub-репозитория нельзя публиковать общий секрет приложения. Поэтому установка из этого репозитория не даёт автоматический доступ к кабинету без настройки учётных данных или собственного OAuth-приложения.

### Хранение секретов

На macOS сервер по умолчанию использует Keychain. На Windows и Linux, а также при явном выборе на macOS, доступно зашифрованное хранилище AES-256-GCM:

```bash
VK_ADS_SECRET_STORE=encrypted-file \
VK_ADS_SECRET_PASSPHRASE='длинный уникальный пароль' \
node dist/index.js
```

По умолчанию файл хранится в `~/.config/vk-ads-mcp/secrets.enc`. Для другого пути задайте абсолютный `VK_ADS_SECRETS_FILE`. Пароль передаётся только процессу сервера и не сохраняется в репозитории.

На macOS токен можно добавить в Keychain командой:

```bash
security add-generic-password -U -a default -s vk-ads-mcp -w "$VK_ADS_TOKEN"
```

Не храните токены и секреты в Git, `secret.md`, `.env` или конфигурации MCP-клиента.

### Несколько кабинетов

Для каждого кабинета используйте отдельный профиль:

```bash
VK_ADS_PROFILE=agency_a VK_ADS_CONNECTION_ID=agency-a npm run dev
```

Секреты получат префикс `agency_a:`. Во время MCP-сеанса нельзя подменить профиль или учётные данные параметрами инструмента.

### Браузерное подключение VK Ads

Инструмент `vk_ads_oauth_begin` открывает официальную страницу запроса доступа. Пользователь выбирает кабинет и подтверждает доступ, после чего сервер получает и сохраняет токены в локальном хранилище.

Этот сценарий требует заранее настроенного приложения VK Ads: его `client_id`, секрет приложения и адрес обратного вызова должны быть зарегистрированы у VK. По умолчанию сервер использует `http://127.0.0.1:39874/`. Разрешены только `localhost`, `127.0.0.1` и `::1` с непривилегированным портом.

```bash
VK_ADS_OAUTH_REDIRECT_URI=http://127.0.0.1:39874/ npm run dev
```

## Сообщества VK

Инструменты статистики и списка управляемых сообществ используют отдельное подключение Core VK. Оно не заменяет доступ к VK Ads. Укажите публичный идентификатор VK ID Web-приложения:

```bash
VK_CORE_VK_CLIENT_ID=идентификатор_приложения
```

Для этого приложения добавьте `localhost` и `http://localhost` в список разрешённых адресов. Затем запустите локальный помощник обратного вызова:

```bash
sudo node /абсолютный/путь/vk-ads-mcp/dist/core-vk-redirect-helper.js
```

После запуска вызовите `vk_core_oauth_begin`, откройте выданную ссылку и подтвердите доступ. Сервер запрашивает только права `groups` и `stats`.

## Безопасная запись

Режим записи выключен. Для тестов включайте его явно:

```bash
VK_ADS_MODE=write node dist/index.js
```

Каждая операция записи проходит четыре шага:

1. Локальная проверка параметров и текущего состояния объекта.
2. Подготовка `write_preview` с точным набором изменений и оценкой риска.
3. Явное подтверждение пользователя одноразовой фразой из preview.
4. Выполнение `write_execute` и повторная проверка результата.

Сервер разрешает создавать и менять только объекты с префиксом `__MCP_TEST__`. Рабочие кампании, группы, баннеры, аудитории, счётчики и бюджеты он не изменяет.

Для загрузки файлов укажите отдельную папку:

```bash
VK_ADS_MODE=write VK_ADS_UPLOAD_DIR=/абсолютный/путь/к/upload node dist/index.js
```

Сервер принимает только обычные файлы из этой папки после проверки фактического пути. Изображения, MP4 и HTML5 ZIP проходят проверку сигнатуры, размера и структуры до создания preview.

Операции с персональными данными и другие рискованные действия требуют отдельных переменных разрешения. Например, для тестовой загрузки списка ремаркетинга нужны `VK_ADS_ALLOW_PII_UPLOADS=1` и `VK_ADS_PII_UPLOAD_DIR`. Текущие ограничения показывает `write_capabilities`.
