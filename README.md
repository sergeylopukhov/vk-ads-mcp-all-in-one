<h1 align="center">VK Ads MCP All in One</h1>

<p align="center">Локальный MCP-сервер для анализа и безопасной работы с VK Ads.</p>

Сервер подключает MCP-клиент к кабинету VK Ads: читает рекламные планы, группы и объявления, получает статистику, работает с аудиториями, медиафайлами и отчётами. По умолчанию запись выключена.

Для Codex, Claude Code, Gemini CLI, Qwen Code и Kimi Code CLI есть примеры подключения по `stdio`. Для других MCP-клиентов может потребоваться отдельная настройка.

> [!IMPORTANT]
> Для анализа публичных сообществ нужен отдельный токен Core VK API: `VK_API_TOKEN`. Он не заменяет учётные данные VK Ads и не применяется к их API.

## Быстрый старт

Нужны Node.js 20 или новее, а также `client_id` и `client_secret` приложения VK Ads.

### macOS и Linux

```bash
curl -fsSL https://github.com/sergeylopukhov/vk-ads-mcp-all-in-one/releases/latest/download/install.sh | sh
```

### Windows

Откройте PowerShell и выполните:

```powershell
irm https://github.com/sergeylopukhov/vk-ads-mcp-all-in-one/releases/latest/download/install.ps1 | iex
```

Установщик запросит `client_id`, скрытый `client_secret`, режим работы и отдельно спросит, нужен ли поиск публичных сообществ VK. Для этой функции по умолчанию предложено встроенное приложение; можно указать своё, если у него есть права `groups` и `wall`. Дополнительные возможности записи настраиваются отдельно. Если установлен Codex CLI, установщик подключит сервер к Codex автоматически.

После установки перезапустите Codex и отправьте запрос:

```text
Покажи контекст подключения VK Ads и доступные рекламные планы. Ничего не меняй.
```

Чтобы обновить сервер, снова выполните команду для своей системы. Установщик сначала покажет установленную и доступную версии, затем предложит: «обновить» или «установить заново». Обновление сохраняет `.env`, профили, токены и локальный аудит и не задаёт вопросов, если обязательные поля уже заполнены. Переустановка заново запрашивает все настройки и заменяет `.env` новыми значениями.

Каталог установки по умолчанию:

- macOS: `~/Library/Application Support/VK Ads MCP`;
- Linux: `~/.local/share/vk-ads-mcp`;
- Windows: `%LOCALAPPDATA%\VK Ads MCP`.

Для Claude Code, Gemini CLI, Qwen Code и Kimi Code CLI используйте [короткие команды подключения](readme/setup-clients.md). Разработчики могут запустить `node install.mjs --help`, чтобы выбрать ветку или другой каталог установки.

## Как создаётся и хранится токен

При первом запросе сервер получает токен VK Ads по `client_id` и `client_secret`, после чего записывает его в локальный `.env` в каталоге установки. Если VK выдаёт `refresh_token`, сервер сохраняет и его для последующего обновления.

`client_secret`, токен доступа и refresh token хранятся только в `.env`. Файл исключён из Git, не попадает в релиз и не нужен в настройках MCP-клиента. Не передавайте его другим людям.

## Разные роли VK Ads: advertiser, agency, manager, ОРД

Права определяет токен пользователя VK Ads, а не сборка MCP. Не используйте один токен для нескольких людей. Для каждого кабинета или роли создайте отдельный локальный профиль:

```bash
cd mcp-server
mkdir -p profiles
cp .env.example profiles/agency.env
open -e profiles/agency.env
```

В файле укажите учётные данные этого пользователя. Затем подключите отдельный MCP-процесс:

```bash
codex mcp add vk-ads-agency --env VK_ADS_PROFILE=agency -- node "$(pwd)/dist/index.js"
```

Профиль `agency` хранит токен и журнал write-операций в `mcp-server/profiles/agency.env` и `mcp-server/profiles/agency.vk-ads-audit.json`. Они игнорируются Git. Так же работают `manager`, `ord_partner` и другие имена профилей. Один MCP-процесс использует один профиль; переключить учётные данные через MCP нельзя.

## Что умеет сервер

| Раздел | Возможности |
| --- | --- |
| Аналитика | Статистика, сравнение периодов, ранжирование по CTR, CPC, CPA и расходу. |
| Реклама | Чтение структуры `ad_plans → ad_groups → banners`; запись выполняется после preflight и явного подтверждения. |
| Креативы | Проверка изображений, видео, HTML5 и параметров объявления до отправки в VK Ads. |
| Данные | Аудитории, сегменты, лид-формы, отчёты и экспорт. |
| Сообщества VK | Поиск, метаданные, анализ публичных постов, скоринг с расшифровкой и экспорт кандидатов в CSV или JSON. Списки участников не запрашиваются. |

## Публичные сообщества VK

При новой установке выберите «Включить поиск и анализ публичных сообществ VK?» и «legacy OAuth». Установщик откроет авторизацию в браузере и попросит полный URL страницы `oauth.vk.ru/blank.html`. Встроенное приложение выбрано по умолчанию. Вместо него можно указать своё приложение, если для него уже доступны права `groups` и `wall`. Также поддерживается VK ID OAuth для ранее настроенных приложений.

Токен Core VK API не заменяет `VK_ADS_TOKEN` и никогда не используется для VK Ads. Он не попадает в MCP-ответы, журнал аудита и диагностические логи. URL callback содержит токен и виден в терминале, поэтому не публикуйте его и не добавляйте в скриншоты. Метаданные хранятся в памяти 10 минут; для Core VK API действует отдельный limiter с повторами при временных ошибках.

Для обычной работы используйте один инструмент: `vk_start_community_research` (или совместимый `vk_research_communities`). Он постранично читает доступную выдачу поиска VK, удаляет дубли и фильтрует metadata. Полный отфильтрованный список сразу сохраняется локально, затем анализируется в фоне пакетами по 25. Пока анализ идёт, сервер отправляет стандартное MCP-уведомление с прогрессом раз в минуту и отдельное уведомление по завершении; в нём есть только счётчики, без токенов, текстов постов и данных рекламного кабинета. Кандидат попадает в анализ только если хотя бы одна поисковая или включающая фраза есть в его названии либо описании; тип, география, размер и исключающие термины также отсекаются до чтения стены. `vk_get_community_research_progress(run_id)` показывает найденные, обработанные и оставшиеся сообщества; `vk_get_community_research_run(run_id)` возвращает готовый снимок. `limit` укажите только если сознательно хотите ограничить объём работы. При `min_members` поиск сначала запрашивается по убыванию числа участников и прекращается после первой страницы, целиком оказавшейся ниже порога. Если конкретный Core VK API-токен возвращает на такой сортировке пустую выдачу, сервер автоматически повторяет запрос по релевантности и продолжает локальную фильтрацию. Если выдача VK или явно заданный лимит не дали охватить всё, в `summary.incomplete_reasons` это будет явно указано. В `summary` есть размер и число пакетов анализа. Результат содержит прошедшие и не прошедшие порог сообщества, настройки, версию скоринга и `run_id`. Снимок хранится локально 30 дней, срок задаётся `VK_COMMUNITY_RESEARCH_TTL_DAYS` от 1 до 90. Экспорт не создаёт аудитории и не добавляет сообщества в сегменты.

Примеры вызовов без записи:

```text
vk_research_communities({"keywords":["настольные игры"],"min_members":1000,"limit":200})
```

Основные параметры:

- `vk_research_communities`: основной инструмент. Принимает поисковые фразы и фильтры, автоматически анализирует активность, сохраняет снимок по `run_id` и отправляет уведомление о ходе раз в минуту, а затем итоговое уведомление. Без `scoring_rules` применяет универсальный профиль: нормированные совпадения в названии, описании и постах, свежесть, долю тематических публикаций, штраф за низкую активность и исключающие термины. Неполные `scoring_rules` меняют только явно заданные параметры; `clusters` добавляют пользовательскую классификацию.
- `vk_get_community_research_progress`: повторно показывает сохранённые счётчики, если MCP-клиент не отображает уведомления.
- `vk_get_community_research_run`: возвращает сохранённый снимок по `run_id` без повторного поиска и анализа.
- `vk_rescore_community_research_run`: пересчитывает сохранённый завершённый запуск по новому профилю без нового поиска, чтения постов или запросов к VK. Создаёт новый `run_id` и сохраняет связь с исходным запуском.
- `vk_find_community_candidates`: совместимый инструмент для одного вызова без сохранения снимка.
- `vk_discover_communities`: низкоуровневый поиск без анализа активности; используйте его, только если хотите самостоятельно собрать цепочку.
- `vk_analyze_communities`: обязательный `community_ids`; доступны `posts_limit`, `analysis_terms` и `exclude_terms`.
- `vk_score_communities`: обязательные `community_ids` и `scoring_rules`. В правилах задаются термины, веса, диапазон размера сообщества, штрафы и минимальный балл; `clusters` — произвольные группы с собственными условиями.
- `vk_export_community_candidates`: принимает результаты поиска или анализа, необязательные результаты скоринга и формат `csv` либо `json`.

Для `vk_score_communities` используйте точную схему. Неизвестные поля и пустые веса отклоняются с ошибкой, а не превращаются в нулевой балл:

```json
{
  "community_ids": [1, 2],
  "scoring_rules": {
    "terms": ["регент"],
    "exclude_terms": ["вакансия"],
    "weights": {
      "name_term": 35,
      "description_term": 15,
      "post_term": 25,
      "activity_fresh": 15,
      "activity_low_penalty": 15,
      "thematic_post_share": 20,
      "members_range": 10,
      "exclude_term_penalty": 30
    },
    "per_match_weights": {
      "name_term": 8,
      "description_term": 3,
      "post_term": 2
    },
    "term_weights": {
      "уставщик": 2,
      "псаломщик": 2,
      "типикон": 2,
      "церковное пение": 1
    },
    "activity_fresh_days": 90,
    "min_posts_per_week": 2,
    "min_thematic_post_share": 0.4,
    "members_range": { "min": 300, "max": 50000 },
    "min_score": 50
  }
}
```

Значения в `weights` — максимальный вклад каждого сигнала: повторные совпадения не могут превысить этот потолок. `per_match_weights` задаёт вклад одного совпадения, а `term_weights` увеличивает его для отдельных профессиональных терминов. `thematic_post_share` добавляет баллы пропорционально доле последних тематических постов; `activity_low_penalty` списывается, если частота публикаций ниже `min_posts_per_week`. Неполные `scoring_rules` переопределяют только явно указанные параметры: базовые веса тематической доли, активности и нормированных совпадений сохраняются. Чтобы отключить сигнал, передайте его вес `0` явно.

Закрытые, удалённые и недоступные сообщества отмечаются флагами. Полные тексты постов не возвращаются и не сохраняются. Поиск, анализ и экспорт не создают аудитории и не меняют рекламный кабинет.

Полный список с описаниями и разделением по категориям опубликован в [TOOLS.md](TOOLS.md). Актуальный набор для запущенного профиля также доступен через MCP-инструмент `search_tools`.

## Режим записи

Для чтения ничего настраивать не нужно. Из папки `mcp-server` запись включается только при явном запуске:

```bash
VK_ADS_MODE=write node dist/index.js
```

В режиме `write` можно подготовить все реализованные операции записи. Перед каждой сервер проверяет данные и создаёт preview; по умолчанию агент выполняет `write_execute` только после явного согласия пользователя именно на это изменение, на любом языке, без ID и фиксированной фразы. Отказ, сомнение, вопрос или нейтральная реплика не являются согласием. Владелец локального профиля может отключить это требование через `VK_ADS_REQUIRE_WRITE_CONFIRMATION=0`; preflight, аудит и предусмотренная повторная проверка сохраняются. Ограничения роли, текущего доступа к объекту и опубликованного контракта VK Ads не обходятся. Перед выполнением проверьте preview и ID.

## Ошибки подключения

- «VK Ads не выдал токен»: проверьте `VK_ADS_CLIENT_ID` и `VK_ADS_CLIENT_SECRET` в `.env`.
- `token_limit_exceeded`: не удаляйте локальный `.env`. При запуске сервер сначала пробует обновить сохранённый `refresh_token`; если VK вернёт эту ошибку, он отзовёт токены текущей связки `client_id—user`, выпустит один новый токен и сохранит `refresh_token`. Для ручного восстановления в write-режиме используйте `vk_recover_token_limit`. Кампании, бюджеты и аудитории не изменяются.
- Ошибка `403`: VK Ads отклонил запрос. Проверьте права приложения и доступ к рекламному кабинету.
