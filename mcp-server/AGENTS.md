# VK Ads MCP

## Контекст

Это самостоятельный локальный MCP-сервер на TypeScript. Он запускается через `stdio` и работает только с VK Ads. `client_id`, `client_secret` и полученный токен лежат в `mcp-server/.env`; не читайте, не печатайте и не добавляйте их в конфигурации клиентов.

Перед изменением кода, тестов, конфигурации, контрактов API или поведения прочитайте `../docs/code_rules.md`, затем `../docs/architecture.md` и профильный документ из карты. Внутренние `../docs/` локальны, игнорируются Git и не входят в релиз.

## Перед вызовом инструментов

1. Вызовите `get_provider_context`, чтобы узнать активный профиль, режим и базовый URL API.
2. Если нужен неизвестный инструмент, найдите его через `search_tools`.
3. Перед работой с рекламным объектом прочитайте его по точному ID. Не подставляйте ID кабинета, кампании, группы или баннера по названию.
4. Для чтения используйте прямой инструмент или `call_read_tool`.

## Изменения в VK Ads

1. Сервер запускается в режиме `readonly`. Режим `write` включают только отдельной переменной `VK_ADS_MODE=write`.
2. Перед созданием или изменением используйте подходящий preflight: `ad_plan_preflight`, `ad_group_preflight`, `banner_preflight`, `creative_preflight` или `media_validate`.
3. Для записи создайте preview через `write_preview` или именованный инструмент preview.
4. Выполняйте `write_execute` только после естественного подтверждения пользователя для свежего preview; явный отказ не подтверждает запись. Владелец профиля может отключить это требование через `VK_ADS_REQUIRE_WRITE_CONFIRMATION=0`.
5. После записи перечитайте объект и сообщите его ID и итоговый статус.

## Защита кабинета

- В `VK_ADS_MODE=write` доступны все реализованные операции записи. Для каждой сохраняются preflight, свежий preview, явное подтверждение и перечитывание результата.
- Проверяйте существование и доступность каждого целевого объекта: ограничения роли и контракта VK Ads остаются фактическими ограничениями.
- Не добавляйте инструменты с произвольным API-хостом, путём, URL, локальным файлом или raw payload.
- Не раскрывайте токены, персональные данные, банковские данные, закрытые URL и содержимое PII-файлов.

## Проверка изменений

После изменения TypeScript-кода запустите из папки `mcp-server`:

```bash
npm run build
```

Внутренние материалы из `docs/` остаются локальными и не входят в GitHub-релиз.
