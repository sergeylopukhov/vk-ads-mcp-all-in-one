# Участие в разработке

1. Не добавляйте токены, client secret, персональные данные, ответы API с такими данными и файлы из локального secret store.
2. Для новых возможностей добавляйте типы, локальную валидацию входных данных и тесты. Записывающие операции должны сохранять read-only режим по умолчанию, preview и одноразовое подтверждение.
3. Перед pull request запустите:

```bash
npm run typecheck
npm test
npm run build
npm run api:check -- --strict
```

4. Если контракт VK Ads не подтверждён официальной документацией или безопасным тестом, отразите это в `vk-ads-capability-matrix.md`.
