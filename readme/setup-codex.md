# Подключение к Codex

Для других MCP-клиентов используйте [общую инструкцию](setup-clients.md).

## Установка

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ> vk-ads-mcp
cd vk-ads-mcp
npm ci
npm run build
```

Нужен Node.js 20 или новее.

## Добавление локального MCP

Выполните из корня репозитория:

```bash
codex mcp add vk-ads --env VK_ADS_PROFILE=default -- node "$(pwd)/dist/index.js"
```

Проверьте регистрацию:

```bash
codex mcp list
```

## Учётные данные

GitHub-репозиторий не содержит и не должен содержать токены или client secret. Настройте один из поддерживаемых способов из [README](../README.md#секреты-и-токены): macOS Keychain, зашифрованный файл вне репозитория или собственный OAuth-клиент VK Ads.

Не добавляйте `VK_ADS_TOKEN`, `VK_ADS_CLIENT_SECRET` или пароль от зашифрованного store в команду `codex mcp add`: Codex сохранит параметры запуска в локальной конфигурации. Для первого подключения используйте безопасное локальное хранилище.

## Первая проверка

В Codex начните с read-only запроса:

```text
Покажи контекст подключения VK Ads и список доступных рекламных планов. Ничего не меняй.
```

Дальше агент должен следовать [AGENTS.md](../AGENTS.md): сначала контекст и чтение, затем preflight, preview, подтверждение и повторная проверка записи.
